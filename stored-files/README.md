# Stored files

Reference docs Scott asks to be saved. Read on demand, never auto-loaded.

## Naming convention

Files are date-prefixed in the format `YYYY-MM-DD-slug.ext` so the newest version is always identifiable in alphabetical order.

Examples:
- `2026-05-04-pipeline-analysis.html`
- `2026-03-04-web-project-non-negotiables.docx`
- `2024-01-24-web-project-welcome-pack.pdf`

When the same source doc is updated, drop the new version in with a fresh date prefix. **Do not delete the old version** — it's the audit trail.

## What goes here

- Source documents that feed `knowledge/frameworks/` and `knowledge/services/` files (the originals; the markdown files reference them)
- Reference docs Scott shares ad hoc and asks to be kept
- Generated outputs Scott wants to keep for later (e.g. one-off analysis HTML)

## What doesn't

- Live operational data (lives in databases, dashboards, MCPs)
- Sensitive credentials, passwords, API keys
- Anything Claude shouldn't read without explicit instruction
