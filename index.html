Part 03

> The 4 Setup Files

Set these up to make Claude work smarter. None are compulsory, but the more you do, the better it gets.

FileWhat It DoesThink of It Like...
CLAUDE.mdYour "house rules" — tells Claude how to behaveAn employee handbook
Context.mdBackground about your business, goals, audienceA consultant briefing doc
MemoryLets Claude remember things across conversationsA CRM / relationship history
SkillsStep-by-step playbooks for specific tasksStandard Operating Procedures
CLAUDE.md

CLAUDE.md

The most important file. Put it in your project's main folder. Write your preferences, do's and don'ts, what the project is about.

"This is a marketing website for [company]. Always write in a friendly, professional tone. Never delete existing content without asking first."
Context.md

Context.md

Gives Claude the "big picture" — what your business does, who your customers are, what stage you're at, key decisions already made and why.

memory/

Memory

Claude's notebook. Just say "remember that I prefer X" and it saves it. Builds up naturally — you don't manage the files yourself.

~/.claude/skills/

Skills

Instruction manuals that make Claude expert at specific tasks. Install once, works across all projects automatically.

Part 07

> Repos, Skills & MCP Servers

Your one-stop shop. Click any item to see what it does. Copy the install command.

Three types of add-ons

Repos = upgrade packs that make Claude smarter overall. Install in Terminal.
Skills = specialist playbooks for specific tasks (SEO, ads, security, etc.). Install in Terminal.
MCP Servers = connectors that plug Claude into apps you already use (Slack, Xero, etc.). Install in Claude Code.

The Universal Install Command

terminal
git clone <github-url> ~/.claude/skills/<name>

Run this once per repo/skill in Terminal. Works across every project on your computer.

Part 08

> Where Everything Lives

The ~ symbol means your home folder (/Users/[yourname]/).

Global settings and skills (shared across all projects):

~/.claude/
  ├── skills/                 <-- All installed add-ons
  │   ├── superpowers/     <-- Example add-on
  │   └── ...
  └── projects/
      └── [project-name]/
          └── memory/      <-- Claude's memory for this project

Your project folder (one per project):

~/Sites/my-project/     <-- Your project folder
  ├── CLAUDE.md         <-- House rules for THIS project
  ├── Context.md       <-- Background info
  └── [your files...]

> Jargon Buster

Plain English translations of terms you'll see.

Common Terms

TermWhat It Actually Means
RepoA folder of files on GitHub — like a shared Google Drive folder
git cloneDownload a repo from GitHub to your computer
TerminalThe app on Mac for typing commands. Find via Spotlight (Cmd + Space)
~ (tilde)Shortcut for your home folder: /Users/[yourname]/
Slash commandTyping /something to trigger a skill (e.g., /commit)
PluginAn add-on for Claude — like browser extensions for Chrome
HookAn "if this, then that" rule for Claude
SkillA pre-written playbook that makes Claude better at a specific task
RepoAn "upgrade pack" of files you can install to extend Claude's abilities

AI & Claude Abbreviations

AbbreviationStands ForWhat It Means
AIArtificial IntelligenceSoftware that can perform tasks normally requiring human intelligence — like writing, coding, analysing, and decision-making

GUIGraphical User InterfaceThe opposite of CLI — windows, buttons, and icons you click. Most apps you use daily

Part 09

> Agents & Agent Teams

How Claude works autonomously, and how to run multiple Claudes in parallel.

What Is an Agent?

An agent is Claude working independently on a task. Instead of you going back and forth message by message, you assign a job and Claude plans the steps, executes them, checks the results, and comes back when it's done or needs a decision.

Regular ChatAgent
How it worksYou ask, it answers, you ask againYou assign a task, it goes away and does it
AnalogyTexting someone for directions step by stepTelling a taxi driver the destination and letting them drive
SupervisionYou guide every stepWorks independently, checks in when needed
You've already been using an agent

This entire session has been agentic. Every time Claude read your files, edited the HTML, ran git commands, and deployed to Vercel — that was an agent at work. You gave the goal, Claude figured out the steps.

Sub-Agents (Workers Within a Session)

Sub-agents are separate Claude instances that the main agent creates for specific jobs. Think of them as temporary workers — they go off, do one thing, and report back with a summary.

how-subagents-work

How They Work

The main agent breaks a task into pieces, assigns each piece to a sub-agent, and they work in parallel. Each one operates independently and sends results back when done. Up to 10 sub-agents can run at once.

when-to-use

When to Use Them

Best for focused tasks where you just need the result — researching multiple topics at once, running tests on different files, or reviewing code from different angles. They can't talk to each other, only back to the main agent.

Agent Teams (The New Feature)

Agent Teams take it further. Instead of workers that just report back, you get a team of Claudes that can talk to each other, share a task list, and coordinate their own work — like a real team.

Sub-AgentsAgent Teams
CommunicationReport results back to main agent onlyTeammates message each other directly
CoordinationMain agent manages everythingShared task list, self-organising
Best forQuick focused tasksComplex work needing discussion
CostLower — results summarised backHigher — each teammate is a full session
AnalogySending out runners to fetch thingsA project team in a room working together

How to Set Up Agent Teams

Step 1: Check Your Version

Agent Teams need Claude Code v2.1.32 or later. Check with:

terminal
claude --version

Step 2: Enable Agent Teams

Agent Teams are turned off by default. You need to enable them. Paste this into Claude Code:

claude-code
Add this to my settings.json: { "env": { "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1" } }

Step 3: Tell Claude to Create a Team

Just describe the task and the team you want in plain language. Claude handles the rest.

example prompt
Create an agent team to work on this project:
- One teammate focused on the homepage design
- One teammate working on the contact form functionality
- One teammate writing the copy for all pages
Have them coordinate and share progress with each other.

Best Use Cases

Use CaseTeam SetupWhy It Works
Code review3 reviewers — security, performance, test coverageEach applies a different lens so nothing gets missed
Debugging3-5 investigators testing different theoriesCompeting hypotheses prevent anchoring on the wrong cause
New featureTeammates each own a separate pieceParallel work without stepping on each other
Cross-layer workFrontend, backend, and test teammatesEach layer gets full attention simultaneously
ResearchMultiple researchers investigating different anglesBroader coverage than one agent going sequentially

Tips for Getting the Best Results

  • Start with 3-5 teammates — more than that and coordination overhead eats the benefit
  • Give each teammate a clear, independent job — avoid two teammates editing the same file
  • 5-6 tasks per teammate keeps everyone productive without them sitting idle
  • Start with research/review tasks if you're new to teams — they have clear boundaries
  • Require plan approval for risky work — say "require plan approval before they make changes"
  • Check in periodically — don't let the team run unattended for too long
  • Use Shift+Down to cycle through teammates and see what they're doing
Cost Warning

Agent Teams use significantly more credits than a single session — each teammate is a full Claude instance with its own context. Use them for complex work where parallel exploration genuinely helps. For routine tasks, a single session is more cost-effective.

The Superpowers Advantage

The Superpowers repo you installed in Part 07 includes a "Subagent-Driven Development" skill that makes agent coordination much more structured — with two-stage review (spec compliance, then code quality) and human checkpoints. It's a great complement to Agent Teams.

> How to Review Claude's Work

You don't need to understand code to be a good reviewer.

"Does It Work?" Check

Show me how to test what you just built. Walk me through it step by step.

"Did It Do What I Asked?" Check

Compare what you just built against my original request. Did you miss anything? Did you add anything I didn't ask for?

"Is It Safe?" Check

Before we go live with this, are there any security concerns? Did you accidentally include any passwords, API keys, or sensitive information in the code?

"Second Opinion" Check

Now pretend you're a senior quality reviewer. Look at everything you just built with fresh eyes. What would you flag as a concern, and what would you improve?
Trust But Verify

Trust Claude for: Routine tasks, first drafts, research, generating ideas.
Verify before approving: Anything going live, involving money, deleting work, or security-related.

> Claude Chat vs Code vs Cowork

Three tools, three different jobs. Using the right one saves time and money.

Claude comes in three flavours. Think of them like this: Chat is like texting a smart friend. Code is like hiring a developer who sits at your computer. Cowork is like hiring an assistant who works on your desktop while you watch.

claude-chat

Claude Chat

The regular chat at claude.ai. You type, it responds. Good for questions, writing, brainstorming, and one-off tasks that don't need your files.

Where: claude.ai (browser) or Claude app

For: Everyone

claude-code

Claude Code

Lives in your Terminal. Can read, edit, and create files on your computer. Runs commands, builds websites, fixes bugs, manages projects. This guide is about this one.

Where: Terminal, VS Code, Desktop app, or browser

For: Anyone working on projects with files

claude-cowork

Claude Cowork

Lives inside the Claude Desktop app. Handles multi-step tasks on your desktop — organising files, creating spreadsheets, building presentations, running research. No coding needed.

Where: Claude Desktop app (Mac or Windows)

For: Non-technical teams who need desktop automation
