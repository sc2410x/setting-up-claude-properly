Part 01

> The 4 Setup Files

Set these up to make Claude work smarter. None are compulsory, but the more you do, the better it gets.

FileWhat It DoesThink of It Like...
CLAUDE.mdYour "house rules" — tells Claude how to behaveAn employee handbook
Context.mdBackground about your business, goals, audienceA consultant briefing doc
MemoryLets Claude remember things across conversationsA CRM / relationship history
SkillsStep-by-step playbooks for specific tasksStandard Operating Procedures
CLAUDE.md

CLAUDE.md

The most important file. Put it in your project's main folder. Write your preferences, do's and don'ts, what the project is about.

"This is a marketing website for [company]. Always write in a friendly, professional tone. Never delete existing content without asking first."
Context.md

Context.md

Gives Claude the "big picture" — what your business does, who your customers are, what stage you're at, key decisions already made and why.

memory/

Memory

Claude's notebook. Just say "remember that I prefer X" and it saves it. Builds up naturally — you don't manage the files yourself.

~/.claude/skills/

Skills

Instruction manuals that make Claude expert at specific tasks. Install once, works across all projects automatically.

> Where Everything Lives

The ~ symbol means your home folder (/Users/[yourname]/).

Global settings and skills (shared across all projects):

~/.claude/
  ├── skills/                 <-- All installed add-ons
  │   ├── superpowers/     <-- Example add-on
  │   └── ...
  └── projects/
      └── [project-name]/
          └── memory/      <-- Claude's memory for this project

Your project folder (one per project):

~/Sites/my-project/     <-- Your project folder
  ├── CLAUDE.md         <-- House rules for THIS project
  ├── Context.md       <-- Background info
  └── [your files...]

> Copy-Paste Prompts

Ready-to-use messages. Copy, paste into Claude Code, press Enter.

Start Every New Project With This

The single most important prompt in this guide. Paste it as the very first thing you say in a new project.

The Recommended First Message

new project
Hi Claude. I've just created this project folder and I need your help setting it up properly and building it out.

Before we do anything, here's what you need to know about me:
- I'm a business decision maker, not a developer
- Always talk to me in plain, non-technical language
- When you make changes, explain what you did and why in terms I can understand
- Don't build anything until we've talked through what I need first
- Ask me questions before making assumptions

Let's start by getting this project set up:

1. Ask me about the project — what it is, who it's for, what I want it to achieve, and any preferences I have. Use these answers to create the CLAUDE.md and Context.md files.

2. Set up your memory system for this project so you remember our conversations.

3. Check that global skills (like Superpowers) are installed and available.

4. Once setup is done, give me a status report and then ask me what I'd like to work on first.

Let's go.

Health Check

any project
Check if this project has the following configuration files set up properly. For each one, tell me: does it exist, where is it, and is it empty or does it have content? Give me a simple status report.

1. CLAUDE.md (or AGENTS.md) — the "house rules" file in the project root
2. Context.md — background info file in the project root
3. Memory system — look for ~/.claude/projects/ folder that matches this project, and check for MEMORY.md inside it
4. Skills — check if ~/.claude/skills/ exists and what's installed there

For anything that's missing or empty, tell me what it should contain in plain, non-technical language. I'm not a developer — I'm a business decision maker.

Format your answer as a simple table:
| File | Status | What to Do |

Get Up to Speed

start of session
Read the CLAUDE.md, Context.md, and your memory for this project. Then give me a brief summary of:
1. What this project is
2. Where we left off
3. What's pending or next

Keep it to a few sentences. I'll tell you what I want to work on today.

End of Session

wrapping up
Before we wrap up, please do the following:

1. Update CLAUDE.md — Add any new rules, preferences, or decisions we made during this session that should apply going forward. Don't remove existing content, just add what's new.

2. Update Context.md — Add any new context about the project, business, audience, or strategy that came up during this session.

3. Update Memory — Save anything important about me, my preferences, or project status that you should remember for next time.

4. Give me a short summary of what was added to each file so I know what changed.

Talk to me in plain language — I'm not a developer.

End of Day Debrief

full day wrap-up
End of day debrief. Please do the following:

1. Review everything we worked on today and summarise the key decisions, changes, and progress made.
2. Update CLAUDE.md with any new project rules or preferences discovered today.
3. Update Context.md with any new business context, goals, or strategic decisions from today.
4. Update Memory with anything you should remember about me, the project, or our working style for future sessions.
5. Create a brief "session log" summary I can refer back to — what we did, what's done, and what's still pending.

Keep everything in plain language. I'm a business decision maker, not a developer.

Remember This

any time
Remember this for all future sessions: [write what you want remembered here]

Explain What Just Happened

when confused
Explain what you just did in plain, non-technical language. I'm not a developer. Tell me:
1. What changed?
2. Why did you do it?
3. What does it mean for the project?
4. Is there anything I need to decide or approve?

Weekly Status Report

once a week
Give me a weekly project status report in plain language:

1. What has been built or changed this week?
2. What's working well?
3. What still needs to be done?
4. Are there any decisions I need to make?
5. Update the Context.md and CLAUDE.md if anything is outdated.

Format it like a business report, not a technical changelog. I'm a decision maker, not a developer.
Part 08

> How to Talk to Claude

The quality of output depends on how you ask. Brief Claude like a freelancer.

Be Specific, Not Vague

Instead of this...Say this...
"Fix the website""The contact form on the homepage isn't sending emails. Find out why and fix it."
"Write some copy""Write a homepage headline for [company]. Audience: small business owners. Tone: friendly, confident."
"Make it look better""The pricing page is cluttered. Simplify layout, add white space, make 'Pro' plan stand out."

Key Principles

  • Tell Claude your role — "I'm a business owner, not a developer. Explain in plain language."
  • Break big tasks into steps — Plan first, build in stages, review at checkpoints.
  • Give examples — "Make it look like [website]" or "Tone like [brand]"
  • Say what you DON'T want — "Don't use jargon", "Don't over-engineer this"

> Common Mistakes

Learn from others so you don't make these yourself.

mistake_01.log

Letting Claude Run Wild Without a Plan

THE MISTAKE

Asking Claude to "build me a website" without reviewing along the way.

THE FIX

Say: "Before you build anything, let's talk through what I need. Ask me questions first."

mistake_02.log

Not Reviewing Before Approving

THE MISTAKE

Clicking "approve" without reading what Claude is about to do.

THE FIX

Ask: "Explain what this will do before I approve it."

mistake_03.log

Not Setting Up Config Files

THE MISTAKE

Jumping into work without CLAUDE.md or Context.md.

THE FIX

Spend 10 minutes using the "First Message" prompt from Part 7.

mistake_04.log

Asking for Too Much at Once

THE MISTAKE

"Build me a complete e-commerce site with accounts, payments, blog, and admin dashboard."

THE FIX

Break it into phases. Start with the most important thing. Get that right, then move on.

Part 10

> When Claude Gets Stuck

Signs: repeating itself, apologising without fixing, suggesting increasingly complex solutions.

Option 1: Reset

Stop. Let's reset. Here's what I need in simple terms: [restate your goal]. What's the simplest way to do this?

Option 2: Break It Down

This isn't working. Let's break this into smaller pieces. What's the first thing we need to figure out?

Option 3: Different Approach

You seem stuck on one approach. Can you think of a completely different way to solve this?
Option 4: Start Fresh

Open a new conversation. Your CLAUDE.md, Context.md, and Memory carry over automatically.

Part 11

> How to Review Claude's Work

You don't need to understand code to be a good reviewer.

"Does It Work?" Check

Show me how to test what you just built. Walk me through it step by step.

"Did It Do What I Asked?" Check

Compare what you just built against my original request. Did you miss anything? Did you add anything I didn't ask for?

"Is It Safe?" Check

Before we go live with this, are there any security concerns? Did you accidentally include any passwords, API keys, or sensitive information in the code?

"Second Opinion" Check

Now pretend you're a senior quality reviewer. Look at everything you just built with fresh eyes. What would you flag as a concern, and what would you improve?
Trust But Verify

Trust Claude for: Routine tasks, first drafts, research, generating ideas.
Verify before approving: Anything going live, involving money, deleting work, or security-related.

> Security Basics

Claude has access to your files. Be smart about what you share.

Never Share...Why Not
PasswordsEnd up stored in files or conversation history
API keysOthers could use your accounts and run up charges
Credit card / bank detailsObvious reasons
Customer personal dataPrivacy regulations (GDPR) and ethics
Keep Secrets in a .env File

A special file that stores passwords/keys locally. Never uploaded or shared. Ask Claude: "Help me set up a .env file for this project."

> Claude Code vs Regular Chat

Different tools for different jobs.

Use Claude Code When...Use Regular Chat When...
Building or editing a websiteAsking general questions
Setting up a projectWriting a one-off email
Fixing a bug in your projectBrainstorming (not tied to a project)
Working with files on your computerStrategy conversations
Need ongoing project contextLearning about a topic
The Simple Rule

If it involves files on your computer or an ongoing project — use Claude Code.
If it's a one-off question — use regular Claude chat.

> Backup Basics

Version control (git) = unlimited undo button for your entire project.

  • Undo anything. Roll back to any previous version.
  • See what changed. Every save has a description.
  • Never lose work. Deleted files are in version history.
  • Try things safely. Experiment knowing you can always undo.

Save a Snapshot

Save the current state of this project as a git commit. Describe what changed.

See What's Changed

Show me what files have changed since the last commit, and summarise the changes in plain language.

Undo / Go Back

Show me the last 5 commits (saved versions) for this project, with a plain language description of each. I might want to go back to an earlier one.
Part 17

> Troubleshooting FAQ

Common problems and how to fix them. Click to expand.

"Claude forgot everything about my project" ▼

Happens when you start a new conversation. Fix: Use the "Get Up to Speed" prompt from Part 7. Keep CLAUDE.md and Context.md up to date.

"Claude won't read my files" ▼

Claude Code needs to be opened from your project folder. Fix: In Terminal: cd ~/Sites/my-project-name then launch Claude Code.

"Skills aren't loading" ▼

Fix: Ask Claude "List everything in ~/.claude/skills/". If there but not working, start a new conversation. If not there, re-run the install command.

"Claude keeps doing things I didn't ask for" ▼

Fix: Add to CLAUDE.md: "Only do exactly what I ask. Don't add extra features or make 'improvements' beyond the task. If you think something else should change, tell me and let me decide."

"The install command failed" ▼
ErrorFix
"repository not found"Search GitHub for the repo name
"destination path already exists"Already installed — no action needed
"command not found: git"Ask Claude: "Help me install git on my Mac"
"I accidentally approved something I shouldn't have" ▼

Fix: If using git: "Undo the last change and go back to the previous version." If not using git — this is why Part 16 (backups) matters.

"I don't understand what Claude just did" ▼

Fix: Paste: "Explain what you just did in plain, non-technical language. I'm not a developer."
